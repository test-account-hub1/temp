감성 브런치 카페 **모닝테이블**의 브랜드 무드와 핵심 정보를 한 페이지에서 전달하는 반응형 랜딩 페이지입니다.

## 프로젝트 소개

모닝테이블은 따뜻한 우드톤과 크림 베이지 기반의 시각 요소를 활용해  
카페의 편안한 분위기를 전달하면서도 방문 전 필요한 핵심 정보를 한 번에 확인할 수 있도록 설계한 랜딩 페이지입니다.

단순히 감성적인 화면 구성에 그치지 않고  
대표 메뉴, 매장 분위기, 공간 정보, 운영 시간, 예약 유도까지 하나의 흐름 안에서 자연스럽게 탐색할 수 있도록 구성했습니다.

## 프로젝트 한 줄 소개

감성 브런치 카페 **모닝테이블**의 브랜드 무드와 핵심 정보를 한 페이지에서 전달하는 반응형 랜딩 페이지

---

## 기획 의도

### 브랜드 의도
- 따뜻한 우드톤과 크림 베이지 기반의 시각 언어로 편안한 아침 이미지를 전달
- 카페의 감성만 강조하지 않고 실제 방문 전 필요한 정보도 함께 제공
- 브랜드 분위기와 실용 정보를 균형 있게 담아낸 랜딩 페이지를 목표로 설계

### 사용자 경험 의도
- 첫 화면에서 브랜드 인상과 행동 유도 버튼을 즉시 제공
- 스크롤 흐름 안에서 정보 탐색이 자연스럽게 이어지도록 섹션 순서를 설계
- 클릭 가능한 상세 정보로 필요한 사람만 더 깊게 확인할 수 있도록 구성

---

## 주요 기능

### 1. 히어로 섹션
- 브랜드 첫인상을 전달하는 메인 비주얼 구성
- 메뉴 보기, 예약 문의 등 행동 유도 버튼 제공
- 매장과 음식 이미지를 활용해 감성적인 분위기 강조

### 2. 대표 메뉴 소개
- 카드형 UI로 대표 메뉴를 한눈에 확인 가능
- 메뉴 정보를 직관적으로 스캔할 수 있도록 구성
- `details/summary`를 활용해 추가 정보를 펼쳐볼 수 있도록 구현

### 3. 매장 및 공간 소개
- 공간의 분위기와 좌석 구성을 시각적으로 전달
- 사용자에게 실제 방문 전 공간 이미지를 상상할 수 있도록 구성

### 4. 운영 정보 제공
- 운영 시간, 위치, 예약 관련 정보를 한 섹션에 정리
- 방문 전에 필요한 실용 정보를 빠르게 확인 가능

### 5. 예약 유도 섹션
- 페이지 하단에서 다시 한 번 예약 문의 행동을 유도
- 정보 탐색 이후 자연스럽게 다음 행동으로 이어질 수 있도록 설계

## 정보 구조

페이지는 아래 순서로 구성되어 있습니다.

**히어로 → 대표 메뉴 → 매장 소개 → 공간 소개 → 운영 정보 → 예약 유도 → 푸터**

이 구조를 통해 사용자는 브랜드 인상을 먼저 받고  
이후 메뉴와 공간, 운영 정보까지 자연스럽게 탐색할 수 있습니다.

## 핵심 구현 포인트

### 1. Sticky Header
스크롤 중에도 주요 메뉴에 즉시 접근할 수 있도록 `sticky header`를 적용했습니다.

### 2. 카드형 UI
메뉴, 공간, 운영 정보를 카드형 패턴으로 통일해  
사용자가 정보를 빠르게 스캔할 수 있도록 구성했습니다.

### 3. 히어로 캐러셀
매장과 음식 이미지를 중심으로 시각적 집중도를 높여  
첫 화면에서 브랜드 무드를 강하게 전달할 수 있도록 구성했습니다.

### 4. 클릭 상세보기
`details/summary`를 활용해 메뉴와 좌석 관련 추가 정보를 필요할 때만 펼쳐서 볼 수 있도록 구현했습니다.

### 5. Hover 인터랙션
버튼, 카드, 이미지에 미세한 hover 반응을 적용해  
정적 페이지에서도 몰입감을 느낄 수 있도록 구성했습니다.

## 문제와 해결

### 문제 1. 감성적인 분위기와 실용 정보를 함께 전달해야 했음
카페 랜딩 페이지는 분위기만 강조하면 실제 방문 정보가 부족해질 수 있고  
반대로 정보만 많으면 브랜드 무드가 약해질 수 있었습니다.

**해결**
- 히어로 섹션에서 브랜드 인상을 먼저 전달
- 이후 메뉴, 공간, 운영 정보 순으로 배치해 감성과 실용성을 함께 담도록 설계

### 문제 2. 한 페이지 안에 정보가 많아질 경우 복잡하게 느껴질 수 있음
여러 정보를 한 화면에 담으면 사용자가 원하는 내용을 찾기 어려울 수 있었습니다.

**해결**
- 상단 네비게이션 앵커 링크 제공
- 카드형 UI 패턴 통일
- 상세 정보는 접어두고 필요할 때만 펼쳐보는 방식으로 구성

### 문제 3. 자바스크립트 없이도 인터랙션이 필요했음
정적인 페이지는 자칫 단조롭게 느껴질 수 있었습니다.

**해결**
- CSS hover 효과
- `details/summary`
- 자동 슬라이드 기반의 시각 연출
등을 활용해 정적인 구조 안에서도 몰입감을 높였습니다.
